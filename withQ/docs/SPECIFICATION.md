# withQ システム仕様書

本ドキュメントは、Discord 募集管理 Bot `withQ` のシステム構成および機能仕様をまとめたものです。

---

## 1. システム概要

`withQ` は、Discord サーバー内でゲームやイベントの参加メンバーを募集するための Discord Bot（スラッシュコマンド対応）です。
ユーザーはコマンドを用いて「募集タイトル」「募集人数」「締め切り時間」などを指定して募集メッセージを作成・送信でき、他メンバーはボタン操作（IN Q, DE Q 等）によってシームレスに参加・取り消しが行えます。

---

## 2. システム構成・動作環境

- **言語・フレームワーク**: Python 3.10+, [discord.py v2.4.0+](https://discordpy.readthedocs.io/)
- **パッケージ**:
  - `discord.py` - Discord API とのインタラクション
  - `Flask` - コンテナを常時稼働（Keep Alive）させるためのダミー Web サーバー
  - `python-dotenv` - `.env` 環境変数管理
  - `pytz` - 締め切り時間計算用タイムゾーン制御
- **コンテナ環境**: Docker / Docker Compose

## 3. コマンド仕様

### 3.1 `/withq`（募集の開始）

募集メッセージを作成して、指定したチャンネルへ送信します。

- **引数**:
  - `募集内容` (必須 / `str`): ゲームタイトルや募集の詳細。
  - `募集人数` (必須 / `int`): 募集主以外の参加枠（1以上の整数）。
  - `締め切り時間` (任意 / `str`): `hh:mm`、`mm/dd/hh:mm`、`yyyy/mm/dd/hh:mm` 形式。未指定時は環境変数で定義されたデフォルト時間が設定されます。
  - `メンション対象` (任意 / `Choice`): `@everyone` または `@here`
  - `通知設定` (任意 / `bool`): 他ユーザーが募集に参加・辞退した際に、募集主へDM通知を送るかどうかのフラグ（デフォルト: `True`）。

### 3.2 `/random`（ランダム選出）

入力された複数の候補値から、1つをランダムで選出して返します。

- **引数**:
  - `candidate` (必須 / `str`): 候補値をスペース区切りで入力。
- **動作**:
  - 入力された文字列をスペースで切り分け、その中からランダムで1つの値を選択してチャネルに送信します。

### 3.3 `/help`（ヘルプ表示）

Bot の使用方法、コマンドのフォーマット、および各インタラクションボタン（IN Q, DE Q, 〆, CANCEL）に関する詳細情報を、入力したユーザー宛て（非公開メッセージ / Ephemeral）で送信します。

### 3.4 `/update`（コマンド同期）

Bot のアプリケーションコマンドツリーを明示的に更新（同期）します。

## 4. 画面インタラクション仕様（募集メッセージのボタン機能）

募集メッセージの下部には以下の4つのボタンが配置されます。

| ボタン表示 | スタイル       | 対象者               | 機能説明                                                                                         |
| :--------- | :------------- | :------------------- | :----------------------------------------------------------------------------------------------- |
| **IN Q**   | Primary (青)   | 全ユーザー           | 募集へ参加します。募集枠が埋まると自動的に募集は締め切られ、全参加者がメンションで通知されます。 |
| **DE Q**   | Danger (赤)    | 参加済みの他ユーザー | 募集の参加を取り消します。募集主自身は取り消せません。                                           |
| **〆**     | Success (緑)   | 募集主のみ           | 任意のタイミングで募集を締め切り、現在の参加者をメンションします。                               |
| **CANCEL** | Secondary (灰) | 募集主のみ           | 募集を取り消して中止します（募集メッセージは取り消し文言に差し替えられます）。                   |

## 5. ビューのライフサイクル・終了制御

募集メッセージ（`RowView`）のライフサイクルおよび終了判定は、Discord.py本来の仕様（ビューのストップ機能）に基づき制御されています。

- **終了検知 (`view.is_finished()`)**:
  募集開始コマンド（`withq`）実行後、バックグラウンドで終了監視ループが動作します。ビューが正常にアクティブなうちは `view.is_finished()` が `False` を返し、ループ内で待機します。
- **終了処理 (`view.stop()`)**:
  以下の場合に `view.stop()` が呼び出され、ビューは非アクティブ状態になり `view.is_finished()` が `True` を返すようになります。
  - 募集主による手動締め切り（**〆** ボタン）
  - 募集主による募集取り消し（**CANCEL** ボタン）
  - 指定された締め切り時間または自動締め切り時間に達した場合（バックグラウンド監視による制御）
- **注意**: `RowView` 内で `is_finished` という名前をカスタムメンバ変数（例：`self.is_finished = False`）として再定義することは禁止されています。親クラス `discord.ui.View` の `is_finished()` メソッドを上書きしてしまい、Discord.py 内部でビュー判定時に `TypeError` が発生するため、必ず組み込みの `stop()` と `is_finished()` メソッドを使用してください。
